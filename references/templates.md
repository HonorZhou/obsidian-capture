# 各来源模板参考

详细 Jinja2 模板见 `assets/` 目录。以下为各来源的正文结构约定。

## 微信公众号深度文章

```markdown
# 公众号名：标题

{正文全文，保留原始 Markdown 格式，分段整理}

## 与你的关联分析

{结合具身智能/深度学习/嵌入式Linux三个方向，逐方向分析本文的关联点和价值}
```

## 微信公众号短讯/早报

按条目罗列，每条 2-3 行概括，不展开深度分析。末尾仍加关联分析。

## 抖音视频笔记

目录结构：
```
01-文章/抖音/{douyin_id}_{标题}/
├── {douyin_id}_{标题}.md    # 主笔记
└── {封面图}                  # 如有
```

主笔记模板：
```markdown
---
title: 视频标题
author: 作者昵称
date: YYYY-MM-DD
type: 抖音
tags: [标签]
source: 原文URL
douyin_id: 数字ID
---

# 视频标题

## 视频信息

| 项目 | 内容 |
|------|------|
| 作者 | {昵称} |
| 发布日期 | YYYY-MM-DD |
| 视频描述 | {原始描述文本} |

## 核心内容

{根据视频描述和评论区整理的核心观点，分段叙述}

## 与你的关联分析

{结合具身智能/深度学习/嵌入式Linux方向进行关联解读}
```
